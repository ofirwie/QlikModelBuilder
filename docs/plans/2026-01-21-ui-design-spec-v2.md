### 2.3 Version Control Integration

Every action creates a **checkpoint**:
- Plan approval â†’ Checkpoint "Plan Approved"
- Chapter completion â†’ Checkpoint "Chapter X Complete"
- Override decision â†’ Checkpoint "Override: [reason]"

**User can:**
- View checkpoint history
- Compare two checkpoints
- Revert to any checkpoint
- Branch from checkpoint (experimental mode)

### 2.4 Detailed Flow per Stage

#### Stage 1: CONNECT
1. Enter Qlik Cloud tenant URL
2. Authenticate via OAuth 2.0 (SSO supported)
3. Select target Space for output app
4. Verify connection with health check
5. **NEW:** Load user's role/permissions for this tenant

#### Stage 2: PLAN
1. Upload/select Stage 1 JSON specification
2. System analyzes tables and relationships
3. System proposes model type with pros/cons
4. System presents execution plan as chapters
5. **NEW:** Team members can comment on plan
6. User reviews and approves plan
7. **NEW:** Checkpoint created: "Plan Approved"

#### Stage 3: BUILD (per chapter)
1. Display chapter goal and expected outcome
2. Generate Qlik script for this chapter
3. **NEW:** Show diff from previous version if exists
4. Execute script on Qlik Cloud
5. Run validation checks
6. Display results with pass/warn/fail indicators
7. **Human Decision Point with Enhanced UI:**
   - âœ… All passed â†’ Approve (creates checkpoint)
   - âš ï¸ Warnings â†’ Decision Dialog with risk explanation
   - âŒ Critical fail â†’ Must address, suggest fixes
8. **NEW:** Checkpoint created after each chapter

#### Stage 4: VALIDATE
1. Full model integrity check
2. Relationship validation (no orphans, no loops)
3. Performance analysis (synthetic keys, cardinality)
4. Gemini AI review of complete script
5. **NEW:** Generate compliance report

#### Stage 5: DEPLOY
1. **NEW:** Deployment Configuration:
   - Target stream selection
   - Overwrite existing app option
   - Schedule data reload
   - Set app permissions
2. Final confirmation with summary
3. Create application in Qlik Cloud
4. Load full data
5. **NEW:** Post-deployment verification
6. Provide direct link to running app
7. **NEW:** Checkpoint created: "Deployed to [Stream]"

## 10. Accessibility (WCAG 2.1 AA)

### 10.1 Compliance Requirements

| Requirement | Implementation |
|-------------|----------------|
| **Color Contrast** | All text meets 4.5:1 ratio (verified in Section 3.2) |
| **Keyboard Navigation** | Full keyboard support, visible focus indicators |
| **Screen Reader** | ARIA labels on all interactive elements |
| **Focus Management** | Logical tab order, focus trapped in modals |
| **Error Identification** | Errors identified by text, not just color |
| **Resizable Text** | UI supports 200% zoom without horizontal scroll |

### 10.2 Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Tab` / `Shift+Tab` | Navigate between elements |
| `Enter` / `Space` | Activate buttons/links |
| `Escape` | Close modals/dialogs |
| `Ctrl+S` | Save current progress |
| `Ctrl+Z` | Undo last action |
| `Ctrl+/` | Open AI assistant |
| `?` | Show keyboard shortcut help |

### 10.3 Color Independence

- Color (visual cue)
- Icon (âœ… âš ï¸ âŒ shape)
- Text label ("Passed", "Warning", "Failed")
