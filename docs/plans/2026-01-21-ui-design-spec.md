# QlikModelBuilder - UI Design Specification

**Date:** 2026-01-21
**Version:** 1.0.0
**Status:** Draft for Gemini Review

---

## Executive Summary

This document specifies the UI design for QlikModelBuilder Stage 2 - a commercial web application that guides Qlik developers through building optimized data models. The application transforms parsed specification JSON into a fully functional Qlik Sense application deployed to Qlik Cloud.

---

## 1. Product Context

### 1.1 Target User
- **Primary:** Junior Qlik Developer (1-3 years experience)
- **Knowledge Level:** Understands Star Schema, dimensions, facts - but needs guidance
- **Work Environment:** Building data models for clients, needs accountability and traceability

### 1.2 Value Proposition
- Transform specification â†’ working Qlik app in hours instead of days
- Built-in validation prevents common data modeling errors
- Human-in-the-loop ensures quality while reducing manual work

### 1.3 Deployment Model
- **MVP:** Cloud-hosted Web Application
- **Future:** Docker container for on-premise deployment
- **Licensing:** Trust-based for MVP, formal licensing later

### 2.2 Detailed Flow per Stage

#### Stage 1: CONNECT
1. Enter Qlik Cloud tenant URL
2. Authenticate via OAuth or API key
3. Select target Space for output app
4. Verify connection with health check

#### Stage 2: PLAN
1. Upload/select Stage 1 JSON specification
2. System analyzes tables and relationships
3. System proposes model type (Star/Snowflake/Link)
4. System presents execution plan as chapters
5. User reviews and approves plan

#### Stage 3: BUILD (per chapter)
1. Display chapter goal and expected outcome
2. Generate Qlik script for this chapter
3. Execute script on Qlik Cloud
4. Run validation checks
5. Display results with pass/warn/fail indicators
6. **Human Decision Point:**
   - âœ… All passed â†’ Auto-continue or manual approve
   - âš ï¸ Warnings â†’ User decides: fix, override, or stop
   - âŒ Critical fail â†’ Must address before continuing

#### Stage 4: VALIDATE
1. Full model integrity check
2. Relationship validation (no orphans, no loops)
3. Performance analysis (synthetic keys, cardinality)
4. Gemini AI review of complete script

#### Stage 5: DEPLOY
1. Final confirmation with summary
2. Create application in Qlik Cloud
3. Load full data
4. Provide direct link to running app

## 4. Color Palette

### 4.1 Design Philosophy
Based on enterprise dashboard best practices:
- **Professional & Trustworthy:** Blue-based primary colors
- **Clear Status Communication:** Semantic colors for pass/warn/fail
- **Accessibility:** WCAG 2.1 AA compliant (4.5:1 contrast minimum)
- **Dark Mode Ready:** Palette supports both light and dark themes

## 6. Interaction Patterns

### 6.1 Decision Points
When user action is required:
1. Dim/blur background content
2. Highlight decision panel with subtle shadow
3. Show clear options with consequences explained
4. Primary action is visually dominant
5. Allow "Learn More" expansion for each option

### 6.2 Loading States
- Skeleton screens for content loading
- Progress bars for long operations (with stage description)
- Disable actions during processing
- Show elapsed time for operations > 5 seconds

### 6.3 Error Handling
- Toast notifications for transient errors
- Inline validation messages for form errors
- Full-page error state for critical failures
- Always provide recovery action

### 6.4 Chat Integration
- Collapsible drawer at bottom
- Context-aware suggestions based on current step
- Maintains history within session
- Can reference specific validation results

---

## 7. Responsive Considerations

### 7.1 Breakpoints
- Desktop: 1280px+ (full sidebar)
- Tablet: 768px-1279px (collapsible sidebar)
- Mobile: < 768px (NOT primary target, basic support only)

### 7.2 Mobile Considerations
MVP targets desktop users primarily. Mobile provides:
- View-only access to project status
- Approve/reject critical decisions
- No script editing capability

---

## 8. Accessibility Requirements

### 8.1 WCAG 2.1 AA Compliance
- Color contrast: 4.5:1 for normal text, 3:1 for large text
- Focus indicators: 2px solid primary color
- Keyboard navigation: Full support
- Screen reader: ARIA labels on all interactive elements

### 8.2 Color-Blind Support
- Status indicators use icons + color (not color alone)
- Charts use patterns in addition to colors
- High contrast mode available

---

## 9. Technology Stack Recommendation

### 9.1 Frontend
- **Framework:** React 18+ with TypeScript
- **Styling:** Tailwind CSS (matches color system)
- **Components:** Shadcn/ui (professional, accessible)
- **State:** Zustand or React Query
- **Code Editor:** Monaco Editor (VS Code engine)

### 9.2 Why This Stack
- Tailwind: Utility-first matches our detailed color specs
- Shadcn/ui: High-quality, accessible, customizable
- Monaco: Professional code editing experience
- React: Large ecosystem, good for complex UIs
